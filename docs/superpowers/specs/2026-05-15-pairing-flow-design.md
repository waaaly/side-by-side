# 配对管理功能设计

## 流程

```
未配对 ──→ 等待中 ──→ 已配对
               ↑          │
               └── 解绑 ──┘
```

## 三个状态

### 1. 未配对 —— Wizard 双入口
- 两张卡片并列：「创建配对」「加入配对」
- 选择后分别进入对应子流程

### 2. 等待中 —— 自动检测伴侣加入
- 大号邀请码 + 复制按钮
- 脉冲动画提示等待
- 使用 Supabase Realtime 订阅自动检测伴侣加入
- 检测到自动跳转到已配对状态

### 3. 已配对 —— 伴侣信息 + 管理
- 双方信息卡片：头像、邮箱、角色（创建者/加入者）
- 解除配对按钮（含确认弹窗）

## 技术要点

- 使用 Supabase Realtime 订阅 pairs 表监听 members 变化
- 创建配对时将当前用户 email 存入 pairs 表
- 加入配对时将双方 email 存入 pairs 表
- 伴侣 email 可在已配对状态展示
